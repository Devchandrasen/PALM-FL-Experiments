# Experiment Repository Audit

This audit describes the experiment-only PALM-FL repository layout.

## Present

| Item | Status | Path |
|---|---|---|
| Core implementation | Present | `palmfl/` |
| Experiment configs | Present | `configs/` |
| Mobile profile builder | Present | `scripts/build_real_mobile_profiles.py` |
| Main experiment runner | Present | `scripts/run_revised_experiments.sh` |
| Ablation runner | Present | `scripts/run_ablation_experiments.sh` |
| Aggregation script | Present | `scripts/aggregate_results.py` |
| Curated trace result script | Present | `scripts/curate_trace_results.py` |
| Diagnostic plotting script | Present | `scripts/plot_experiment_figures.py` |
| Package consistency check | Present | `scripts/check_results_consistency.py` |
| Curated CSV summaries | Present | `analysis/*.csv` |
| Trace-derived mobile profile | Present | `data/mobilebandwidth/real_mobile_profiles.csv` |

## Not Included

| Item | Reason |
|---|---|
| Dataset caches | Regenerated by torchvision loaders. |
| Raw output folders | Can be large; regenerated under `outputs/`. |
| Non-experiment deliverables | This repository keeps only code, configs, CSV summaries, and experiment utilities. |
| Real-device power traces | Not collected in this package. |

## Open Experiment Extensions

- Full multi-seed CIFAR-10 operating-mode and baseline reruns.
- Real-device latency or energy calibration.
- Additional heterogeneous FL baselines.
- Attack-oriented leakage diagnostics for released summaries.
